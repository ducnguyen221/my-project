1. Định nghĩa, Ý nghĩa & Giá trị

🎯 Company/ là gì?

DucNguyen Company OS — hệ điều hành công ty cho mô hình "1 founder + N AI agent" hoạt động như team 10 người. Đây là single source of truth cho mọi hoạt động kinh doanh: từ engagement với khách, cohort COMPA Class, deal pipeline, đến code product và brand voice.

💎 Giá trị cốt lõi (4 trụ)

  1. Single source of truth — mỗi knowledge có duy nhất 1 nơi sống. Tool khác chỉ là bridge import. Loại bỏ drift.
  2. Compound knowledge — mỗi engagement/cohort/campaign sinh asset reusable, lưu vào playbook. Năm sau tốc độ x2 mà không tăng cost.
  3. AI-augmented operator — 1 người + 23 AI agent persona = năng suất team 10 người. AI đóng vai SDR/Consultant/Course Designer/PM... khi Đức cần.
  4. Cross-tool resilient — đổi tool (Cowork → tool X năm sau) không cần rebuild. Knowledge sống ở Layer 1 độc lập.

🆚 Vì sao không dùng Notion / Google Drive thuần?

❌ Notion / GDrive

  • Format proprietary — AI agent đọc qua MCP, mất 30-50% structure
  • Vendor lock — export sang đâu cũng pain
  • Không có hierarchical CLAUDE.md auto-load
  • Search yếu hơn grep filesystem
  • Không Git versioning ngon

✅ File system + Markdown

  • AI đọc thẳng — Cowork/Claude Code/AG đều mount filesystem
  • Markdown plain = future-proof, không bao giờ chết
  • CLAUDE.md auto-load mọi level (Anthropic convention)
  • grep/glob nhanh — search bất kỳ keyword
  • Git version control + private repo

2. Cách dùng & Quy trình AI Agent

🔄 Working Loop chuẩn (mọi agent đều theo)

Read → Plan → Implement → Verify → Update-back → Report

🎬 Khi nào AI agent đọc Company/ ?

1
Khởi tạo session Cowork/Claude Code: tool tự mount thư mục → đọc CLAUDE.md root → đọc .claude/settings.json → đọc MEMORY.md → load context.
2
Khi Đức gõ slash command (vd /daily): tool tìm .claude/commands/daily.md → execute theo workflow trong file đó.
3
Khi Đức nói "đóng vai SDR": Cowork match keyword → load sales/.agent/agents/sdr.md → agent persona kích hoạt.
4
Khi event nghiệp vụ xảy ra (tạo file meeting, save content marketing): hook trong .claude/hooks/hooks.json tự trigger script.
5
Khi user request chuyên sâu (vd "viết blog Power BI banking"): Cowork match skill description → trigger skill (vd anthropic-skills:blog-writer) → đọc .claude/output-styles/compa-class-blog.md để format.

🧭 Load order chuẩn

1. CLAUDE.md (root)              ← entry point cho mọi tool
2. .claude/settings.json          ← permissions, model, MCP allowlist
3. MEMORY.md (root)              ← master memory index
4. .company/CLAUDE.md             ← rule cấp công ty (Tier 0)
5. <dept>/CLAUDE.md              ← rule cấp phòng ban (khi user vào dept đó)
6. <file nghiệp vụ>               ← context cụ thể của task

🔍 Cơ chế discovery trong knowledge base lớn

Tầng 1Hierarchical CLAUDE.md auto-loadMỗi session, mặc định

Tầng 4Glob + Grep on demandKhi user hỏi cụ thể
